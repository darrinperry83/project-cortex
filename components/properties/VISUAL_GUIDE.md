## Size Reference

All chips use:

- **Text size**: `text-xs` (12px) or `text-sm` (14px)
- **Padding**: `px-2 py-1` (8px horizontal, 4px vertical)
- **Border radius**: `rounded-full` or `rounded-lg`
- **Gap between chips**: `gap-2` (8px)

## Accessibility Features

1. **Keyboard Navigation**:
   - Tab through form fields in editor
   - Enter to save
   - Escape to cancel

2. **ARIA Labels**:
   - Delete buttons have aria-label="Delete property"
   - Close button has aria-label="Close"

3. **Focus Management**:
   - Key input auto-focuses on open
   - Focus returns to trigger after close

4. **Screen Reader Support**:
   - Proper labels for all inputs
   - Error messages announced
   - Loading states indicated

## Animation & Transitions

- **Chip hover**: Smooth background color transition (200ms)
- **Delete button**: Fades in on hover using `opacity-0` â†’ `opacity-100`
- **Modal**: Backdrop blur with fade-in
- **Dialog**: Smooth entrance/exit with Radix Dialog animations

## Best Practices

1. **Keep property names short**: Long keys wrap awkwardly
2. **Use consistent naming**: snake_case or camelCase
3. **Group related properties**: Use similar prefixes (e.g., `date_opened`, `date_visited`)
4. **Limit properties per block**: 5-10 max for readability
5. **Use appropriate types**: Don't store numbers as strings
6. **Taglist for categories**: Better than multiple boolean flags

## Future Enhancements Ideas

1. **Property Presets**: Quick-add common property sets
2. **Property Search**: Filter blocks by property values
3. **Bulk Edit**: Edit same property across multiple blocks
4. **Property Inheritance**: Child blocks inherit parent properties
5. **Custom Colors**: User-defined colors for property types
6. **Property Sorting**: Reorder properties via drag-and-drop
7. **Property History**: Track changes over time
8. **Property Formulas**: Computed properties based on others
